- [ ] `p3` Scope `font-feature-settings` away from the global `body` selector. Currently `body { font-feature-settings: "ss01", "cv11"; }` applies Lexend's stylistic alternates to everything, which is harmless today (Tailwind `tabular-nums` uses `font-variant-numeric`, not `font-feature-settings`, so no clobber; and no `<code>`/`<pre>` in the app). But it's latent fragility: any future child element that sets `font-feature-settings` directly will silently wipe `ss01`/`cv11` because the property doesn't compose across the cascade ‚Äî it replaces. Two fixes: (a) scope to Lexend-using elements (`body, input, button, textarea, select`), or (b) use `font-variant-alternates: styleset(...) character-variant(...)` with a `@font-feature-values Lexend { ... }` rule for semantically-correct, composable application. (b) is the idiomatic modern answer
